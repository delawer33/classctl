# Руководство пользователя classctl

## Запуск

```bash
classctl
```

Откройте браузер: http://127.0.0.1:8000

---

## Настройка класса

Перед первым использованием необходимо создать класс.

**1. Нажмите «+ Новый»** в левой панели.

**2. Заполните поля:**

- **Название** — произвольное имя класса (например, «Зал 101»)
- **Подсеть (CIDR)** — подсеть класса (например, `192.168.10.0/24`)
- **Путь к SSH-ключу** — путь к приватному ключу (например, `/home/user/.ssh/id_rsa`)
- **Пользователь SSH** — имя пользователя на рабочих станциях (по умолчанию: `student`)
- **Каталог скриптов** — путь к папке со скриптами на рабочих станциях (например, `/home/student/VBox_install`)
- **Таймаут WoL** — сколько секунд ждать SSH после включения машин (по умолчанию: 300)

**3. Маппинг шагов** — укажите имена файлов скриптов для каждого шага:

| Шаг | Действие |
|-----|----------|
| 1 | Остановить ВМ |
| 2 | Удалить ВМ |
| 3 | Сбросить хост |
| 4 | Создать ВМ |
| 5 | Выключение (опционально) |

**4. Нажмите «Создать».**

---

## Управление машинами

### Обнаружение машин

Нажмите **«↻ Обновить»** — приложение выполнит ARP-сканирование подсети и добавит найденные машины в список. Ноутбук должен быть подключён к подсети класса.

Новые машины подсвечиваются меткой **«новая машина»**.

### Ручное добавление

Нажмите **«+ Добавить»**, введите IP и MAC-адрес машины.

### Удаление машины

Нажмите **«✕»** напротив нужной машины.

### Выбор машин для запуска

Используйте чекбоксы слева от каждой машины. Кнопки **«Выбрать все»** и **«Снять выбор»** работают для всего списка.

---

## Запуск пайплайна

**1. Выберите начальный шаг** — пайплайн запустится с выбранного шага и пройдёт до шага 4.

**2. Wake on LAN** — если машины выключены, оставьте галочку включённой. Если машины уже работают — снимите.

**3. Нажмите «▶ Запустить»** — появится запрос на подтверждение с указанием шага и списка машин.

**4. Подтвердите запуск.**

После запуска панель конфигурации сворачивается и открывается монитор выполнения.

---

## Монитор выполнения

Во время работы пайплайна отображается таблица со статусом каждой машины:

| Статус | Значение |
|--------|----------|
| ОЖИДАНИЕ | Машина ещё не начала выполнение шага |
| ВЫПОЛН. | Скрипт выполняется |
| ОК | Шаг завершён без ошибок |
| ОШИБКИ | В выводе найдены строки, совпадающие с шаблонами ошибок |
| ТАЙМАУТ | Скрипт не завершился за отведённое время |
| РАЗРЫВ | SSH-соединение прервано во время выполнения |
| ПРОПУСК | Машина пропущена оператором |

Нажмите **«▼»** справа от машины, чтобы раскрыть полный вывод скрипта.

В шапке отображается текущий шаг и прошедшее время.

Если машина не выдаёт вывод более 2 минут, в колонке «Последний вывод» появится подсказка с временем молчания — это нормально для длинных скриптов.

---

## Пауза и принятие решений

Если после завершения шага обнаружены ошибки (статусы ОШИБКИ, ТАЙМАУТ или РАЗРЫВ), пайплайн приостанавливается и появляется баннер с тремя вариантами действий:

- **↩ Повтор** — повторить текущий шаг на проблемных машинах
- **⟶ Пропустить** — пометить проблемные машины как пропущенные и перейти к следующему шагу
- **✕ Прервать** — остановить пайплайн полностью

> ⚠ При статусе РАЗРЫВ: скрипт может всё ещё выполняться на машине. Повтор запустит новый экземпляр скрипта.

---

## Выключение машин

**Выключить выбранные вручную:** нажмите **«⏻ Выключить выбранные»** — команда `shutdown` будет отправлена на все отмеченные машины по SSH.

**Автоматически после завершения пайплайна:** установите галочку **«Выключить после завершения»** перед запуском.

Машины, участвующие в активном запуске, не будут выключены принудительно.

---

## Настройки

Нажмите **«Настройки»** в правом верхнем углу.

**Шаблоны обнаружения ошибок** — список строк (без учёта регистра), при наличии которых в выводе скрипта шаг считается завершённым с ошибками. По умолчанию: `error`, `failed`, `traceback`, `exception`.

Добавьте, удалите или измените шаблоны и нажмите **«Сохранить»**.
